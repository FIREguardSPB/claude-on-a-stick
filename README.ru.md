# claude-on-a-stick

> 🇬🇧 English version: [README.md](README.md)

Публичный кросс-платформенный интерактивный **сборщик** (лицензия MIT), который
превращает обычную USB-флешку в портативное окружение **настоящего Claude Code**
без установки - на вашей собственной подписке. Токен авторизации хранится на флешке
в зашифрованном виде, есть антибан-геозащита (geo-guard) и опциональный
встроенный VPN на базе Happ.

В репозитории лежит **только логика**. Сам бинарь Claude, Happ, токен авторизации
и ссылку на подписку загружает/вводит сам пользователь **на этапе сборки**, после
чего они оказываются на флешке, но **никогда не попадают в git**.

---

## Одна флешка - любая ОС

**Одна USB-флешка запускает Claude Code на Windows, Linux *и* macOS - из одного
зашифрованного токена.** На этапе сборки выбираете одну целевую платформу или сразу
несколько (список через запятую, либо `A` - «все основные» = `win32-x64` + `linux-x64`
+ `darwin-arm64`). Сборщик скачивает и проверяет по sha256 бинарь под каждую платформу
в отдельную подпапку, а лаунчеры сами определяют, какой бинарь запустить на той машине,
куда вы воткнули флешку:

```
<STICK>/
  bin/win32-x64/claude.exe     # по одной подпапке на каждую собранную платформу
  bin/linux-x64/claude
  bin/darwin-arm64/claude
  config/  oauth.enc  settings.json  .claude.json   # ОДИН зашифрованный токен, общий для всех ОС
  projects/   tmp/   (лаунчеры ...)
```

Токен авторизации и `config/` пишутся **один раз** и общие для всех ОС - один и тот же
`oauth.enc` расшифровывается и на Windows (PowerShell), и на Linux/macOS (openssl). При
запуске скрипт `env` вычисляет `CLAUDE_BIN` - бинарь под текущую ОС и архитектуру
(`x64`/`arm64`, плюс glibc-или-musl на Linux). Сборка под одну ОС работает ровно как и
раньше - по умолчанию целевая платформа просто та, на которой вы собираете.

## Что это такое

`claude-on-a-stick` собирает «флешку с Claude»: вставляете её в любой компьютер,
запускаете один файл (`START.bat` на Windows, `start.sh` на Linux/macOS) - и
получаете рабочий Claude Code, который не устанавливается в систему и не оставляет
следов на хост-машине. Вся конфигурация, проекты и временные файлы живут на самой
флешке.

Ключевые свойства:

- **Без установки.** Claude - это самодостаточный бинарь (Node внутри не нужен),
  он запускается прямо с флешки.
- **Ваша подписка.** Используется ваш собственный долгоживущий токен (на данный момент 1 год)
  (`claude setup-token`, только для инференса), зашифрованный вашим паролем.
- **Геозащита от банов.** Перед запуском проверяется страна выхода в интернет; из
  заблокированных регионов запуск либо идёт через встроенный VPN, либо отменяется.
- **Опциональный VPN.** При желании на флешку кладётся портативный Happ в
  прокси-режиме (без прав администратора, без TUN).

## Честные пределы (читайте до сборки)

Мы не приукрашиваем. Вот что работает надёжно, а что - экспериментально.

- **Windows и Linux - надёжно, проверено.** Сборка и запуск на этих ОС протестированы:
  формат AES-токена сверен в обе стороны, скачивание бинаря и проверка контрольной
  суммы работают, USB-форматирование и лаунчеры отлажены.
- **macOS - экспериментально, «по мере сил».** Mac для проверки не было. Сборщик
  печатает предупреждение «experimental» и предлагает ручные запасные шаги
  (Gatekeeper/quarantine, кодподпись, прокси-режим без повышения прав - всё это
  не проверено вживую).
- **VPN - опционален.** Если ваш регион не заблокирован, VPN не нужен и **не
  включается** - геозащита его не трогает. Это и есть смысл «умного пропуска».
- **Авто-вставка подписки - направляемая (guided), а не полностью автоматическая.**
  Happ - это SingleApplication: ссылка передаётся уже запущенному экземпляру через
  `happ://`-deep-link, который импортирует подписку сам. Если что-то пошло не так,
  сборщик **печатает ссылку**, и вы импортируете её в Happ вручную один раз.

## Быстрый старт (по ОС)

Сначала **соберите флешку** (см. ниже «Как идёт сборка»), затем запускайте её на
любой машине.

### Windows

1. Вставьте флешку.
2. Откройте её в проводнике и запустите **`START.bat`** (двойной клик).
3. Введите **пароль флешки** (Stick password), когда попросят, - он расшифрует токен
   в память.
4. Откроется Claude Code, рабочая папка - `projects/` на флешке.

Диагностика: запустите `DIAG.bat`. Включить VPN отдельно: `vpnup.bat`.

### Linux

1. Вставьте флешку и смонтируйте её (обычно монтируется автоматически).
2. В терминале перейдите на флешку и запустите **`./start.sh`** (при необходимости
   `chmod +x start.sh`).
3. Введите **пароль флешки** (Stick password) - токен расшифруется в память.
4. Откроется Claude Code, рабочая папка - `projects/`.

Диагностика: `./diag.sh`. Включить VPN отдельно: `./vpnup.sh`.

### macOS (экспериментально)

1. Вставьте флешку.
2. В терминале запустите **`./start.sh`**.
3. Если macOS заблокирует Happ или бинарь по карантину, снимите атрибут вручную:
   `xattr -dr com.apple.quarantine <путь>` (сборщик подскажет конкретный путь).
4. Введите **пароль флешки** (Stick password).

> На macOS часть шагов не проверена. Следуйте подсказкам сборщика и запасным
> инструкциям, которые он печатает.

## Как идёт сборка (флоу)

Сборщик интерактивный и запускается на той ОС, под которую (или на которой) вы
делаете флешку:

- **Linux / macOS:** `builders/posix/build.sh` (нужен root для форматирования USB):
  `sudo ./builders/posix/build.sh`
- **Windows:** `builders/windows/build.ps1` (полноценный, первого класса -
  аудитория на Windows скачивает и запускает именно его)

**Windows: запуск сборщика.** По умолчанию Windows блокирует выполнение `.ps1`
(`running scripts is disabled on this system`). Запускайте от имени
**администратора** (нужно для форматирования USB) одним из двух способов; оба
обходят запрет только для этого скрипта:

- **Проще всего:** в папке `builders\windows` щёлкните **`run-build.bat`** правой
  кнопкой -> «Запуск от имени администратора» (или просто двойной клик). Обёртка
  запускает `build.ps1` с обходом политики выполнения.
- **Либо напрямую** в окне PowerShell с повышенными правами:
  ```powershell
  powershell -ExecutionPolicy Bypass -File .\builders\windows\build.ps1
  ```

Порядок шагов сборщика:

1. **Язык.** Сначала спрашивается язык интерфейса (RU/EN), значение по умолчанию
   берётся из системной локали; переопределяется одной клавишей.
2. **Целевая платформа(ы).** По умолчанию - та машина, на которой вы собираете (одна
   платформа). Для мультиплатформенной флешки введите список через запятую (например,
   `win32-x64,linux-x64,darwin-arm64`) или `A` - «все основные». Доступны: `win32-x64`,
   `win32-arm64`, `linux-x64`, `linux-arm64`, `linux-x64-musl`, `linux-arm64-musl`,
   `darwin-x64`, `darwin-arm64`.
3. **Выбор и форматирование USB.** Сборщик показывает только съёмные USB-диски
   (внутренние и системные диски не предлагаются никогда), печатает id + размер +
   модель и требует ввести подтверждение (`ERASE`/`YES`) с повтором id устройства.
   Затем создаётся **одна** partition: MBR, тип `0x07`, файловая система **exFAT**
   с меткой `CLAUDE`. Форматирование выполняется **один раз** независимо от числа
   выбранных платформ.
4. **Скачивание Claude.** С официального хоста релизов резолвится версия (канал
   **stable** по умолчанию, опция - latest), для **каждой** выбранной платформы
   скачивается свой бинарь и **проверяется по sha256**; при несовпадении сборка
   прерывается. Каждая платформа кладётся в свою подпапку `bin/<platform>/`. Если в
   системе есть `gpg`, дополнительно проверяется подпись манифеста.
5. **Токен (один раз, общий для всех ОС).** Вы вводите свой токен
   (`claude setup-token`) и **пароль флешки**. Токен шифруется (AES-256-CBC, ключ из
   PBKDF2-HMAC-SHA1, 300000 итераций) и кладётся на флешку как `config/oauth.enc`. Этот
   же зашифрованный токен читают все ОС, под которые собрана флешка.
6. **Модель.** Выбор модели по умолчанию (по умолчанию `claude-opus-4-8`), значение
   «запекается» в лаунчер как `--model`.
7. **VPN (опционально).** Для флешки под одну ОС: на флешку кладётся портативный Happ в
   прокси-режиме; можно вставить ссылку на подписку - сырой URL (превратится в
   `happ://add/<urlenc>`) или готовый `happ://crypt5/…` дословно. Для мультиплатформенной
   флешки VPN по умолчанию **выключен** (бинари Happ привязаны к ОС и большие, ~300 МБ
   каждый); при согласии под каждую ОС кладётся своя копия в `apps/happ-<os>/`. Без VPN
   геозащита опирается на хостовый/системный VPN.
8. **Лаунчеры.** На флешку копируются стартовые скрипты (`START`/`start`, `DIAG`,
   `env`, `vpnup`, `geoguard`, `decrypt`), шаблонизированные под выбранные ОС, язык
   и модель. Для мультиплатформенной флешки набор лаунчеров - **объединение**: набор
   Windows, если выбрана хоть одна `win32-*`, и набор POSIX, если выбрана хоть одна
   Linux/macOS платформа.

Что попадает на флешку:
`bin/<platform>/claude(.exe)` (по подпапке на каждую платформу),
`config/{oauth.enc,settings.json,.claude.json}` (общие), опционально `apps/happ-<os>/…`,
плюс папки `projects/`, `tmp/` и сами лаунчеры. При запуске `env` сам выбирает нужный
бинарь (`CLAUDE_BIN`) под текущую ОС и архитектуру.

### Сборка за прокси / в ограниченном регионе

Сборщик скачивает официальный бинарь `claude` с `downloads.claude.ai`. В
ограниченных регионах этот хост может не резолвиться без прокси или VPN. Сборщик
направляет **весь** свой сетевой трафик на этапе сборки (манифест, версия,
загрузка бинаря, GPG-подпись, Happ) через прокси, если он доступен:

- **Задайте явно.** Перед сборкой укажите переменную окружения с локальным
  HTTP-прокси вашего VPN:
  - Windows (PowerShell): `$env:HTTPS_PROXY = "http://127.0.0.1:10809"`,
    либо передайте `-Proxy http://127.0.0.1:10809` в `build.ps1`.
  - Linux/macOS: `export HTTPS_PROXY=http://127.0.0.1:10809`.
- **Или автоопределение.** Если прокси не задан, сборщик пробует
  распространённые локальные порты HTTP-прокси (10808, 10809, 2080, 2081, 1080,
  10800, 7890, 8080, 1087) и берёт первый рабочий - просто включите VPN в режиме
  **прокси** и запустите сборку.
- **Или системный прокси.** Системный прокси Windows учитывается автоматически.

Замечания: для сборщика Windows работают только прокси вида `http(s)://` (адрес
`socks5://` пропускается с уведомлением; curl на Linux/macOS умеет и SOCKS). Если
хост по-прежнему недоступен и прокси не найден, сборщик выводит понятное
сообщение с просьбой включить VPN или задать `HTTPS_PROXY` и повторить.

## Модель безопасности

Главное, что нужно понять заранее:

- **Шифруется только токен.** На флешке в зашифрованном виде лежит **только** файл
  `config/oauth.enc` (ваш токен авторизации). Расшифровка происходит в память при
  запуске после ввода пароля флешки; расшифрованный токен **никогда не пишется на
  диск** (передаётся Claude через переменную окружения `CLAUDE_CODE_OAUTH_TOKEN`).
- **Транскрипты и работа - в открытом виде.** Ваши проекты, история диалогов и
  временные файлы в `projects/` и `tmp/` лежат на флешке **незашифрованными**.
  Шифрование токена их не защищает.
- **Для персональных данных (ПДн) клиентов - шифруйте весь том.** Если на флешке
  будут чувствительные данные, используйте полнодисковое шифрование самого тома:
  **BitLocker To Go** (Windows), **VeraCrypt** (кросс-платформенно) или **LUKS**
  (Linux). Подробности - в `docs/SECURITY.md`.

Криптоформат (для тех, кому нужны детали): файл `oauth.enc` имеет вид
`salt[16] || iv[16] || AES-256-CBC(PKCS7)-шифртекст`; ключ выводится через
PBKDF2-HMAC-SHA1, 300000 итераций, 32 байта. SHA1 в KDF выбран ради совместимости
между openssl, LibreSSL и .NET.

## Юридическая заметка

- **Бинари не распространяются.** В этом репозитории **нет** ни бинаря Claude, ни
  Happ. Они загружаются с официальных источников **на этапе сборки** и попадают
  только на вашу флешку. Репозиторий содержит исключительно логику сборки.
- **Токен - ваш собственный.** Используется ваш личный токен подписки
  (`claude setup-token`), который вы вводите сами; он шифруется вашим паролем. Мы
  не «минтим» подписки через сторонние сервисы и ничего не передаём третьим лицам.
- **Лицензия - MIT.** Код сборщика открыт под MIT.
